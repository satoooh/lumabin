# AI開発者向けガイドライン

## 言語ルール

- ドキュメント: 日本語
- ログ・コード内コメント・コミットメッセージ: 英語

## コミット規約

[Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) に従う。 scopeも必ず含める。

### 形式

```
<type>(<scope>): <description>

- 変更内容1
- 変更内容2
```

### ルール

- 1行目: `<type>(<scope>): <description>` の形式
- 本文: 箇条書きで変更内容を説明（任意だが推奨）

## ドキュメントの参照

コードの変更や調査を行う際は、必ず先に docs/AGENTS.md のドキュメント一覧を確認し、関連するドキュメントを読むこと。ドキュメントにはシステムの設計思想・アーキテクチャ・規約が記載されており、これらを理解してから作業を行うことで、既存の設計と矛盾しない変更ができる。

## コード修正時のチェックリスト

コードを修正した際は、以下を必ず確認すること:

1. 関連する README.md の更新が必要か確認
2. 関連する AGENTS.md の更新が必要か確認
3. 各サブプロジェクトのビルド・テストが通ることを確認

## 開発実行の安定運用（Electron / UI検証）

`apps/desktop` の実装・検証時は、以下を標準手順として扱う。

1. 開発起動前に、残存 `agent-browser` daemon を停止する（FD枯渇回避）。
2. 同じシェルで `ulimit -n` を確認し、低い場合は引き上げる（目安: `4096` 以上）。
3. `npm run dev` は one-shot 実行ではなく、常駐セッション（TTY）で維持する。
4. 実機UI確認が必要な変更（レイアウト/スクロール/モーダル/フォーカス）は、スモークテストだけで完了扱いにしない。
5. UI変更時の最低検証順序は `smoke:ui` → `smoke:ci` とし、失敗時は修正後に再実行する。
6. 検証後は不要な常駐プロセスを停止し、次作業への影響を残さない。

## plan 作成時のルール

実装計画を作成する際は、コード変更だけでなく、以下のドキュメント更新もスコープに含めること:

1. 関連する README.md の更新
2. 関連する AGENTS.md の更新
3. その他の関連ドキュメント（docs/、PRD など）の更新

## AGENTS.md の記述方針

軽微なコードの修正で AGENTS.md の整合性が失われることがないように、AGENTS.md は自然言語や図で、ロジックや構造などに注目した説明をする。
具体的なファイルパスやコードの詳細ではなく、設計意図や規約を記述する。
