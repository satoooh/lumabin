# LumaBin Product Brief

- 最終更新日: 2026-05-04
- ステータス: Public preview

## 1. プロダクト定義

LumaBin は、Cloudflare R2 上の画像・動画・PDF・CSV を、gallery-first の体験で管理する macOS デスクトップアプリ。

**一番重要な一文**:

> LumaBin は「オブジェクトストレージ管理ツール」ではなく、
> 「R2 上の視覚アセットを気持ちよく扱う作業環境」である。

## 2. ターゲット

- Cloudflare R2 を日常運用する個人開発者
- Web 制作者 / デザイナー / コンテンツ運用者
- S3 概念は理解しているが、日常操作は GUI で完結したいユーザー

## 3. 解く課題

既存の汎用 S3 クライアントは「転送中心」であり、視覚アセット運用の導線（探す・見る・選ぶ・共有）が重い。
LumaBin はこの導線を最短化し、R2 を日常運用できる体験を提供する。

## 4. 設計原則

### 4.1 美しいこと

- 見たら分かる UI を優先する
- 説明文より操作の自己説明性を重視する

### 4.2 最小限

- 同じ価値なら機能数・設定数が少ない案を選ぶ
- 新機能追加より既存導線の磨き込みを優先する

### 4.3 最高のデフォルト

- 初期設定のまま主要フローを完走できる
- 追加設定は「既定値で解決できない理由」がある場合のみ導入する

### 4.4 モードレス優先

- 一覧と詳細を分断せず、連続した作業体験として設計する
- 画面遷移は「別画面へ飛ぶ感」より「同一空間での拡大/縮小」を優先する
- 状態切替時も文脈（選択・位置・対象）を保持し、再学習コストを発生させない

## 5. 提供価値

- 迷わない接続（R2 first）
- gallery/list と高速検索による探索
- drag & drop 中心のアップロード体験
- image/video/pdf/csv の即時プレビュー
- URL共有（public / presigned）

## 6. 差別化ポイント

1. **Gallery first**: ファイル名リストではなく視覚選択を主軸にする
2. **Workflow first**: 転送機能ではなく運用体験（探索/仕分け/共有）を主軸にする
3. **Local intelligence**: ローカル索引とキャッシュで R2 のレイテンシを UX で吸収する

## 7. スコープ

### 7.1 含む

- 接続プロファイル（R2 / generic S3）
- 一覧、検索、フィルタ、Saved Views、Smart Collections
- アップロード（競合解決、retry/cancel、multipart）
- プレビュー（image/video/pdf/csv）
- 基本操作（rename/move/delete/bulk）
- 共有（public URL / presigned URL）

### 7.2 含まない

- マルチユーザー権限管理
- Web 公開ポータル
- AI 自動タグ付け
- Finder 拡張 / ドライブマウント

## 8. 固定判断

- 名前（仮）: LumaBin
- TS-only 方針を維持
- Electron 採用
- R2 first / S3 ready
- 競合ポリシー既定は Rename
- Smart Collections をローカル状態で提供

## 9. 成功条件

- 接続→一覧→アップロード→プレビュー→共有を説明なしで完了できる
- `smoke:ci` と `package` が安定して通る
- 既知の P0/P1 バグがクローズされている
