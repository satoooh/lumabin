## 1. 現在地（要約）

- `apps/desktop` は日常利用可能な Beta 状態（接続/一覧/検索/アップロード/プレビュー/共有まで成立）
- CI は `desktop-ci`、E2E監視は `desktop-e2e`、配布導線は `desktop-release`（tag 起点の zip 配布）で運用中
- gallery-first + modeless 遷移を維持しつつ、UIの単純化を継続
- キャッシュ・索引・アップロード失敗耐性の hardening は一巡完了
- 次は「体感品質の詰め」と「保守しやすい分割」の2軸で改善を進める
- release readiness の要求別証跡と未完了事項は [RELEASE_READINESS_AUDIT.md](RELEASE_READINESS_AUDIT.md) に分離して管理する

### パフォーマンス

- スクロール中の重い処理を削減（`onScroll` の仕事量を縮小）
- UI state 永続化をデバウンスし、連続操作時の負荷を軽減
- `scroll-behavior: smooth` の常時適用を外し、ホイールスクロールの引っ掛かりを低減
- gallery/list のスクロール state 更新を `requestAnimationFrame` で制御し、更新頻度を抑制
- gallery スクロール中はサムネイル取得を一時停止し、描画スパイクを低減
- gallery スクロール中はカード hover/zoom の transition を抑止し、dense bucket のスクロール体感を改善
- gallery virtual overscan と scroll 更新閾値を調整し、トラックパッド連続操作時の再描画負荷を低減
- preview close 後のフォーカス復帰時に list/gallery それぞれへ安全に追従スクロールするよう調整
- 低幅時に inline feedback を自動非表示化し、主要操作導線の優先表示を維持
- 低幅時は compact query summary を自動退避し、主要操作コントロールの視認性を優先
- gallery サムネイル loading state 更新を batch 化し、連続読み込み時の明滅を抑制
- スクロール中は gallery の loading shimmer を停止し、ちらつき体感を低減

- UI smoke の keyboard flow テストを安定化（`activeElement` 同一ノード比較を廃止し、選択コンテキスト検証へ変更）

## 6. 既知リスクと監視ポイント

- Renderer workbench の肥大化による保守コスト（bounded context ごとの workbench / presenter 分割が必要）
- DDD/CQRS/EDA 境界は Main Process の runtime composition 分離が一巡したため、次は Renderer workbench と view model 分割を継続する必要がある
- UI導線の増加に対して自動回帰が追いつかないリスク
- provider 差分（R2 と generic S3）に起因する edge case
- GitHub Actions の action major update 後に、runner 側の仕様差分が workflow へ影響する可能性
- full history には過去のprivate profile / image host / private email が残るため、そのまま repository visibility を public に変更しない。公開する場合は sanitized snapshot / squash / 履歴 rewrite のいずれかを選ぶ
- GitHub 側に active Actions artifacts と non-draft Releases が残る間は、既存 repository の visibility を public に変更しない
